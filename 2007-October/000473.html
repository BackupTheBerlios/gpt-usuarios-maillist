Esse código me lembrou que Z, um identificador, não está no plural.
Então, temos o seguinte contraste:

a : matriz[1] de inteiros;
b : matriz[2] de Pessoa;

Pergunta: o parser deve ser inteligente o suficiente para reconhecer
as regras de plural para as palavras?

-Z -> Zs
-Pessoa -> Pessoas
-Ser -> Seres
-Identificação -> Identificações

Acho que as regras são simples. O mais complicado, talvez seja o caso
"PESSOA", mas poderiamos fazer uma pequena análise do case do
identificador, para dedizirmos "PESSOAS".....

Talvez seja interessante. O que acham?
