Você não entendeu porque eu não fui claro rsrsrs.

Vamos a um exemplo melhor. Vou assumir que o nosso int tem 16 bits no
exemplo, o que não será verdade na VM.

Caractere, no meu entendimento, pode variar de 0 a 255. Int varia de 0 a
65536 (16 bits).

Ambos são armazenados como um int.

Agora, se tivermos o seguinte código em portugol:

algoritmo exemplo;
variáveis
    c: caractere;
    i: inteiro;
fim-variáveis
início
    c := 255;
    i := 255;
    c := c + 1; // c deve ser 0 agora
    i := i + 1; // i deve ser 256 agora
fim

Com isso eu quero dizer que, apesar de ambos serem armazenados como um
inteiro, algumas operações tem que ser específicas conforme o tipo de dado.

O código acima geraria o seguinte código em GptAssembly:

program exemplo;

var c char // internamente será um int no bytecode e durante a execução da
VM
var i int

proc main
    iset c, 255 // iset tanto para caractere como inteiro
    iset i, 255
    csum c, c, 1 // csum converte para char antes de somar
    isum i, i, 1 // isum soma diretamente como um inteiro
end-proc

end-program
