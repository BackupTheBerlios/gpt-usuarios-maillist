Oi pessoal.

Como o Thiago comentou em outro e-mail, o ideal seria termos suporte a
funções/procedures com número de argumentos (parâmetros) variáveis.

Para uma procedure com número fixo de argumentos, quando esta procedure for
invocada em algum ponto do código, o compilador gera o seguinte código em
assembly:

// Código em GPT
...
procedure p1 (x: inteiro, y: inteiro)
...

...
p1(10,20);
...


// Código em GptAssembly
push 20 // empilha argumentos na ordem inversa
push 10
pcall p1



Para lidar com o caso de número de argumentos variáveis eu teria a seguinte
proposta...

Sempre que o compilador for invocar uma função/procedure que tenha o número
de argumentos variável, o compilador teria que, além de empilhar cada
argumento, empilhar também o tipo de cada argumento e o número total de
argumentos.

// Código em GPT
...
procedure p2 (x: inteiro, ...) // chutei "..." como a forma de especificar
que tem argumentos variáveis....
...

...
p2(1, 10, "20", 30.00);
...


// Código em GptAssembly
push 30.00 // empilha argumentos na ordem inversa
push real
push "20"
push string
push 10
push int
push 1 // não precisa tipo porque tá explicito na definição da procedure
push 4 // número de argumentos total, inclusive os não variáveis
pcall p1


Esse metodologia serviria também para o caso da procedure "imprima" de forma
que ela não seja mais tratada como "um caso especial".

Sugestões ? Reclamações ? Dicas ???
