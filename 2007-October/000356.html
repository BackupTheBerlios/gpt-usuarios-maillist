Fazendo um rollback mental, então...

Vamos esquecer o antlr com python. Para o compilador oficial, acho que
todos concordam que não é uma dependência interessante.

O que nos resta, então, é o antlr v3 com C, ou antlr v2 com C++.

Eu fiz uns poucos testes aqui com o v3/C e UTF-8 ontem, e a coisa
pareceu funcionar sem maiores problemas. Mas, eis minhas impressões
iniciais:

1) O código C não é dos mais bonitos (nomeclatura, estilo, etc)...mas
td bem, eu tenho estômago...

2) Falta documentação do target C. Acho que tem api gerada pelo
doxygen, mas não tive tempo de dar uma lida. Algumas coisas básicas eu
tive que sacar por tentativa e erro, inspecionando o código gerado.

3) Ainda não consegui entender e utilizar direito o mecanismo de
excessão. Pelo que me lembro, essa é uma parte importante do
compilador e quero estar certo de que está funcionando bem (afinal, C
não tem excessões).

Apesar de tudo, pelo menos os exemplos de código são abarrotados de
comentários explicando o que está acontencendo.

Mas senti falta de uma boa documentação do próprio antlr. Talvez eu
esteja com problemas em navegar no wiki, então não vou assumir muita
coisa por enquanto. Mas estou achando difícil encontrar explicações
para vários recursos.

No mais, estou esperando meu livro chegar (sim, comprei o livro do antlr).

Quanto a tradução dirigida pela sintaxe, não sei. Talvez possamos
utilizá-la para gerar código rapidamente, mas no fim das contas, acho
que será interessante termos a AST, especialmente para fazermos coisas
como o fluxograma do algoritmo do usuário, de forma rápida.
