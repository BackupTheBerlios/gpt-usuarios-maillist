On 10/16/07, Alex Garzao <alexgarzaol em gmail.com> wrote:
> > > > == Matrizes ==
> > Esse assunto todo me fez parar para pensar melhor sobre tipos e
> > matrizes. Cheguei a seguinte proposta:
> >
> >
> > Para as declarações:
> >
> > a : matriz[2] de inteiros;
> > b : matriz[2][2] de inteiros;
> >
> > Entendemos os tipos de "a" e "b" como:
> >
> > a -> *(inteiro)
> > b -> *(*(inteiro))
> >
> > Onde * denota uma coleção, e *(T) denota uma coleção de valores do
> > tipo T. Note que, aqui, o tamanho das dimensões não faz parte do tipo.
> >
> > Então, poderiamos declarar mais umas variáveis:
> >
> > c : matriz[5] de inteiros;
> > d : matriz[] de inteiros;
>
> Matriz sem tamanho definido, a essas alturas do campeonato, complica um
> pouco as coisas pelos motivos que conversamos em outro e-mail...

Ok. Então fica como plano para depois, pq acho que isso permitira a
flexibilidade que estamos procurando...

> > E seus tipos seriam:
> >
> > c -> *(inteiro)
> > d -> *(inteiro)
> >
> > Sabendo que uma atribuição só pode ser feita para tipos compatíveis,
> > as seguintes atribuições são válidas:
> >
> > a := c;
> >   -> *(inteiro) := *(inteiro) // ok
>
> a tem 2 elementos e c tem 5. O que aconteceria nesse caso ???

O mesmo que aconteceria se 'a' e 'c' fossem literais, e 'a' tivesse 2
caracteres e 'c' tivesse 5 caracteres.

> > d := b[1]
> >   -> *(inteiro) := *(inteiro) //ok
>
> d não tinha um tamanho inicial, e nesse caso vai ter que ser alocado durante
> a  execução...

Uma cópia de 'b[1]' será referenciada por 'd'. Da mesma forma que uma
atribuição de literais.

> > Também, uma vez que um valor do tipo *(T) tenha tamanho N, ele não

> A questão não é se ele muda ou não de tamanho, mas se ele tem um tamanho

> dado, é como se ele tivesse mudado de tamanho (de zero para alguma coisa).

Sim, assim como literais também. A variável apenas passa a referenciar
