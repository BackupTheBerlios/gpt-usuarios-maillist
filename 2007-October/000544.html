O único problema que vc observou da abordagem 2 foi o limite de
dimensões em uma matriz? Acho que se tivermos m1,...,m5 não teremos
grandes problemas.

Eu pensei em outra abordagem aqui. Não é tão limpa, pois diverge mais
ainda o tratamento de dados, mas consegue abstrair totalmente as
operações, sem gerar mais opcodes.

Não pensei sobre suas limitações em casos especificos, nem se é viável
para nós, mas vou colocar meus pensamentos, apenas pela discussão (e
perdoem qualquer falha na explicação, especialmente, nos exemplos):

[Abordagem extra]

Suponha que uma matriz de uma dimensão (e, possivelmente, uma
estrutura) seja apenas uma tripa de área na memória contigua (um
array), e isso é tudo o que a VM sabe a respeito. Não há opcodes para
trabalhar com estes dados, e variáveis são apenas ponteiros para estas
áreas.

Uma matriz[3] de inteiros seria uma área contigua onde cada célula
amazenaria um dos inteiros.

Uma matriz[3] de literais seria uma área contigua capaz de armazenar 3
ponteiros, que referenciariam os respectivos literais. O mesmo
aconteceria para matrizes de estruturas e matrizes n-dimensionais (que
seriam apenas arrays que apontam para arrays ....)

De qualquer forma, uma matriz[10] de inteiros ocupa o mesmo espaço de
uma matriz[10] de literais e de uma matriz[10] de estruturas.

A única diferença de matriz[10] de inteiros das demais matrizes é que
esta pode ter cada célula com o próprio valor, e não com um ponteiro
para o valor.

Toda a manipulação desta memória ficaria a cargo de funções escritas
no runtime, e chamadas utilizando libcall. Poderiamos definir que o
formato da área de memória de uma matriz e de uma estrutura tenha um
cabeçalho contendo informações sobre o dado, como tamanho e tipo, e
funções do runtime poderiam obter estas informações.

Exemplo de uso:

//gpt
m : matriz[10] do tipo inteiro;
m[0] := 1

//gptasm
push m
push 0
push 1
libcall gptruntime.so, "mset"

//gptruntime.c

gpt_coringa mset(void* matriz, gpt_inteiro idx, gpt_coringa valor) {
 //seta o "valor" para matriz[idx]
}


//Exemplo 2
//gpt
m : matriz[10][5] do tipo literal;
x : literal;
x := m[1][2];

//gptasm
var t1 int //t1 é um ponteiro. declara-se como int mesmo?

push m
push 1
libcall gptruntime.so, "mget"
pop t1           //t1 aponta para uma matriz matriz[5]

push t1
push 2
libcall gptruntime.so, "mlitget"
pop x

//gptruntime.c
gpt_coringa mget(void* matriz, gpt_inteiro idx) {
  //retorna o valor em matriz[idx]
}

gpt_literal mlitget(void* matriz, gpt_inteiro idx) {
 //retorna o literal apontado por matriz[idx]
}
