On 10/16/07, Alex Garzao <alexgarzaol em gmail.com> wrote:
> Só para sintonizar, na api do antlr (v2), o tratamento de erros era através
> de excessões ?
>

Isso. Na do v3, também. Por exemplo, MismatchedTokenException é
lançado quando se espera um token e o lookahead não satisfaz.
NoViableAltException é lançado quando não há produções que possam
consumir o lookahead. E como cada produção é um método, é possível
capturar uma excessão lançada em uma produção aninhada. Coisas super
legais :)

No V3, para o target C, cada excessão é identificada por um valor
numerico definido com #define. Mas ainda não sei como é feito a
"desempilhação" de chamadas.
