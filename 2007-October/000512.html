Oi pessoal.

A princípio os tipos primitivos da VM serão mapeados para os tipos da
arquitetura em que está rodando. Um inteiro será um int32 ou int64 (conforme
a arquitetura).

Porém essa abordagem está me parecendo nada portável...

Como fica a questão da portabilidade do gvm com os tipos primitivos ? Se o
gvm foi gerado em uma arquitetura de 32 bits, cada inteiro foi gerado com 32
bits. Mas quando a VM ler esse gvm em uma arquitetura de 64 bits ??? Se a
gente usar um int nativo, a VM 64 bits vai ler cada int como 8 bytes...

Para resolver isso eu pensei em armazenar no gvm o tamanho dos tipos
primitivos dependente de arquitetura. Dessa forma a VM, quando executar a
carga do arquivo, teria condições de mapear os dados corretamente. Só que
isso daria um pouco de trabalho, e não seria uma solução perfeita...

Outra forma seria a gente definir o que é o nosso int. Por exemplo, o nosso
int tem 32 bits em qualquer arquitetura. Isso facilitaria muito as coisas
para nós em vários pontos. Porém, também não é uma solução perfeita. Se uma
lib do SO retornar um dado de 64 bits, o que aconteceria ??? Como mapear
isso ???
