Oi pessoal.

Estou começando a formular algumas idéias sobre como a VM conseguirá
utilizar as libs do sistemas (syslibs, neste e-mail).

Primeiramente eu gostaria de salientar alguns problemas relevantes que
devemos resolver:

1) Temos que, de alguma forma, mapear uma chamada a uma biblioteca na GptVM
para uma chamada a uma syslib;

2) Não temos como, para cada chamada, implementarmos um código para tal "na
munheca". Temos que ter em mente que serão muitas (mas muitas mesmo)
chamadas de sistemas que teremos que mapear. Para mim fica claro que teremos
que automatizar isso;

3) As syslibs utilizam tipos de dados que não necessariamente são
compatíveis com os tipos da GptVM, ou seja, temos que mapear os tipos para a
syslib e o retorno para a GptVM;

4) Não necessariamente uma chamada a uma syslib será simples como apenas
traduzir o nome da função e pronto. Possivelmente parâmetros podem estar
implícitos e não serem passados durante a chamada da syslib.

Exemplo: funcao abre_arquivo invoca a syslib fopen com nome do arquivo
apenas. Como o arquivo será aberto é um valor default passado a fopen que
não é passado durante a chamada a abre_arquivo.

Certamente existem vários outros problemas que nem imaginei. De qualquer
forma estou tentando bolar uma proposta inicial.

A minha primeira idéia é criar um gerador de bindings para a GptVM (já
conversamos sobre isso na lista antes). Esse gerador recebe um arquivo
contendo dados como nome da lib a ser gerada, header (.h) utilizados e a
definição de cada uma das funções. Com esses dados o gerador cria os .hpp e
.cpp contendo basicamente o mapeamento entre nome de funções em Portugol e
syslibs, além de fazer as devidas conversões nos tipos de dados dos
parâmetros e retorno de função.

Exemplo de um arquivo de definição da lib "test":

$ cat test.sld
// sld = syslib definition
option {
   libname   = "test",
   baseclass = "CBaseSyslib",
   linkerlib = "math"
}

header {
   "stdlib.h",
   "math.h",
   "iostream"
}

procedure mensagem_padrao()
{
   puts("Hello !!!")
}

// Quando o mapeamento é bem simples o exemplo abaixo poderia ser uma
"otimização" para o usuário digitar menos...
procedure mensagem_personalizada(cstring msg) ::= puts(msg);

function get_mensagem() : string
{
   string msg;
   std::cin >> msg;
   return msg;
}

function exponencial(double valor) : double
{
   return exp(valor);
}

A partir do arquivo acima o gerador de bindings criaria três arquivo: .hpp,
.cpp e o Makefile.

Abaixo vou listar o que seria o .hpp e o .cpp.

$ cat syslib_test.hpp
#include <"stdlib.h">
#include <"math.h">
#include <"iostream">

class CSyslib_test : public CBaseSyslib
{
public:
   static void mensagem_padrao(Parameters parameters, Return return);
   static void mensagem_personalizada(Parameters parameters, Return return);
   static void get_mensagem(Parameters parameters, Return return);
   static void exponencial(Parameters parameters, Return return);
}


$ cat syslib_test.cpp

void CSyslib_test::mensagem_padrao(Parameters parameters, Return return)
{
   puts("Hello !!!");
}

void CSyslib_test::mensagem_personalizada(Parameters parameters, Return
return)
{
   char* msg = parameters.getParString(0);
   puts(msg);
}

void CSyslib::get_mensagem(Parameters parameters, Return return)
{
   std::string msg;
   std::cin >> msg;

   return.setString(msg);
}

void CSyslib::exponencial(Parameters parameters, Return return)
{
   double valor = parameters.getParDouble(0);

   valor = exp(valor);

   return.setDouble(valor);
}
