Pensando alto....

Se mapearmos "long" ou 'long long' para um "inteiro", os bindings
farão a tradução, cortando o valor "inteiro" que vem da VM para o tipo
C esperado pela função, se esta recebe um int, por exemplo. Pensando
rapidamente, acho que isso é aceitável.

Mas se mapeamos um "int" para "inteiro" (como é hoje) e se um binding
tem uma função que retorna um "long", acho que temos um problema.
Cortar o "long" mapeando-o para "inteiro" não me parece legal. A
última opção que pensei foi em mapear o long retornado para uma
estrutura específica, retornando-a. O problema disso é que o usuário
não vai poder somar, subtrair, etc o valor utilizando os operadores +,
- etc. O binding teria que fornecer funções portugol para fazer essas
operações. E isso tudo, tbm não me soa bem.
