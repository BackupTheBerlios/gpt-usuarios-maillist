Até o CI, sem problemas, porque os comandos do CI independem do tipo de
dado.

Porém, para a VM, o tipo de dado importa. Para converter para bytecode,
temos
que ter um set, get, add, sub, mul, div, ... para cada tipo de dado.

Para o tipo inteiro, teríamos as seguintes instruções (tentando gerar
instruções
equivalentes aos comandos do CI):

* setic (inicializa variável inteira com uma constante) para instruções como
x := 10, y := 20, ...
* setiv (inicializa variável inteira com o conteúdo de outra variável) para
instruções como x := y, z := t, ...
* addicc (soma duas constantes inteiras) para instruções como x := 10 + 20,
...
* addivv (soma duas variáveis inteiras) para instruções como x := y + z, ...
* addivc (soma variável e constante) para instruções como x := y + 10, ...
* addicv (soma constante e variável) para instruções como x := 10 + y, ...
* + 4 instruções para sub, 4 para mul, 4 para div e 2 para neg.

Essas instruções foram apenas para o tipo int. Ainda temos os tipos real,
string, char e bool.

É isso que eu quis dizer com "inchado". Para refletir o CI diretamente sobre
o bytecode,
só inchando o número de opcodes, o que eu acho que não seria legal...

O que os processadores "reais" fazem (e acredito que várias VM's também)
é não deixar as possíveis operações (no nosso caso, opcodes) tão "flexíveis"
assim.

Para minimizar um pouco a gente poderia converter instruções como v1 := v2 +
v3 para algo como:

setiv v1 v2
addiv v1 v3

Instruções como v1 := 10 + v2 para algo como:

setic v1 10
addiv v1 v2

Instruções como v1 := v2 + 10 para algo como:

setiv v1 v2
addic v1 10

Instruções como v1 := v2 para algo como:

setiv v1 v2

Instruções como v1 := 10 para algo como:

setic v1 10

Note que nestes exemplos eu utilizei praticamente as mesmas instruções em
todos os exemplos (setiv, setic, addiv e addic).

Além disso talvez seja possível reduzir ainda mais alguns opcodes. Tenho
algumas
idéias mas tenho que pensar nisso "com calma"...

Não sei se consegui esclarecer as coisas ou piorar hehehe. O meu cérebro tá
"batendo pino" hoje hehehe.
