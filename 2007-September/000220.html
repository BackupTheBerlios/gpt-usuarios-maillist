Algumas notas: para o v2 eu gostaria de ter as funções imprima() e
leia() implementadas em portugol (fazendo uso de código inline em
baixo nível para acesso ao kernel). Dessa forma, elas precisam ser
revistas.

Primeiro, devemos decidir se imprima() continuará recebendo uma lista
variável de argumentos. Se sim, devemos definir a sintaxe e semântica
para funções que utilizam parâmetros variáveis. Caso contrário,
possibilitar overriding de funções baseado nos tipos de argumentos
seria suficiente, na minha opnião:

  função imprima(i : inteiro) ...
  função imprima(s : literal) ...
  função imprima(r : real) ...
  ...

  imprima("2 + 2 = ");
  imprima(2 + 2);
  imprima("\n");
  //Imprime: 2 + 2 = 4

Para resolver a legibilidade, poderiamos utilizar um operador de
concatenação capaz de operar sobre qualquer tipo de dados. Então:

  imprima("2 + 2 = " >> (2 + 2) >> "\n"); //O operador >> é apenas ilustrativo


No caso da função leia(), que atualmente é um absurdo semântico, ela
se beneficia, automaticamente, da possibilidade de utilizar passagem
por referência:

  função leia(ref i : inteiro) ...
  ...

  leia(x);

Se adotarmos overriding, seria melhor ainda. Teriamos:

  leia(nome);
  leia(idade);

Ao invés de sermos obrigados à:

  leia_literal(nome);
  leia_inteiro(idade);

Agora, se levarmos em consideração a utilização de bibliotecas
