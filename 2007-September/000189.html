Oi pessoal.

Em outro e-mail eu havia dito que a nossa expectativa inicial era que o CI
fosse o assembly da nossa VM. Eu também havia dito que, após vários testes,
eu não tinha conseguido esse resultado, e estava optando por gerar um
assembly para termos mais um passo intermediário entre o portugol e o
bytecode.

Dessa forma a gente tinha as seguintes etapas:

Portugol -> Transformado em AST -> Gera CI -> Gera assembly -> Gera bytecode

Essa estrutura, como comentei antes, é bem similar a estrutura do GCC.

Neste final de semana, após vários testes (papel apenas, nada de código), eu
consegui aproximar os opcodes da VM a tal ponto que é quase trivial gerar
bytecode a partir do CI. Dessa forma não existe mais a necessidade de gerar
um assembly intermediário.

Porém, na minha opinião, fica muito estranha a estrutura do compilador sem o
assembly. Sempre que visualizo essa idéia tenho a sensação de que "está
faltando alguma coisa".

Se possível eu gostaria de saber a opinião de vocês.

Segue abaixo um exemplo em portugol e o programa equivalente em CI (CTE no
nosso caso) e assembly. As únicas diferenças entre o CI e o assembly é que,
ao gerar o assembly, é explicitado as funções conforme o tipo de dado e
traduzido as operacções para o mnemônico correto da VM. Claro, para gerar o
bytecode, seria necessário um montador assembly, mas esse montador deve ser
relativamente trivial porque ele basicamente só traduziria os mnemônicos
para o opcode correto, faria o endereçamento correto de variáveis e labels,
e geraria o arquivo contendo o bytecode.

// Exemplo em Portugol

algoritmo exemplo;
início
    imprima( "2+5*3=", 2+5*3 );
fim

// Exemplo em CTE

program exemplo

proc main
    var t1 int
    var t2 int

    // imprima( "2+5*3=", 2+5*3 );
    push "2+5*3"
    pcall imprima
    t1 := 5 * 3
    t2 := 2 + t1
    push t2
    pcall imprima
    preturn
end-proc

// Exemplo em Assembly

program exemplo

proc main
    var t1 int
    var t2 int

    // push "2+5*3"
    push "2+5*3"

    // pcall imprima
    pcall imprimas

    // t1 := 5 * 3
    imul t1 5 3

    // t2 := 2 + t1
    iadd t2 2 t1

    // push t2
    push t2

    // pcall imprima
    pcall imprimai

    // preturn
    preturn

end-proc
