É, mas pelo menos dá pra fazer (em tese) os "calculos quaisquer"
gerando números randômicos. Claro que toda vez que o programa rodar,
os mesmos números serão gerados...hehe. Mas mesmo assim, problemas que
eu ignorei no passado começam a aparecer. Por exemplo, tentei portar
uma implementação do basicão Linear congruential generator [1] e
obtive isso:

//--codigo
algoritmo rand;

variáveis
  randd_xn:inteiro;
fim-variáveis

início
 randd_xn := 0xA673F42D;
 teste();
fim

função randd():real
 r:real;
início
  randd_xn := 1103515245 * randd_xn + 12345;
  r := randd_xn;
  retorne r / 4294967295;
fim

função teste()
 i : inteiro;
início
 para i de 0 até 100 faça
   imprima(randd());
 fim-para
fim
//--codigo


Ai apareceram dois problemas que eu joguei por debaixo do tapete esses
anos todos: a representação dos números não foi muito bem pensada no
gpt. Primeiro que rodar esse algoritmo compilado e interpretado geram
valores diferentes. Segundo que eu não consegui portar o algoritmo do
seguinte programa C adequadamente.

unsigned int randd_xn = 0xA673F42D;

double randd(void)
{
	randd_xn = 1103515245 * randd_xn + 12345;
	return (double) randd_xn / UINT_MAX;
}

int main() {
  int i = 0;
  for (i = 0; i < 100; i++) {
    printf("%f\n", i, randd());
  }
}


[1] - http://en.wikipedia.org/wiki/Linear_congruential_generator
