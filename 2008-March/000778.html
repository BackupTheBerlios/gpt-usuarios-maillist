Hoje brinquei um pouco no Windows, tentando adaptar algum editor para
usar com o GPT. Infelizmente, não tive muito sucesso. Pelo menos, não
como eu esperava. Na lista, estão inclusos o Notepad++, JEdit,
TextPad, Crimson e Programmers Notepad.

-JEdit: não consegui encontrar, por padrão, nenhuma opção para
executar comandos no console (nem mesmo plugins).
-TextPad: não é free, e não encontrei suporte para utf-8.
-Crimson: não é free, não consegui executar comandos satisfatoriamente.
-Programmer's Notepad: não consegui executar comandos satisfatoriamente.

O único editor que pareceu suportar o que eu esperava, de forma
simples,  foi o notepad++. Criar sintaxe colorida para a linguagem foi
tranquilo (o que eu não esperava, devido à linguagem ter acentos).
Porém, o arquivo onde a definição criada da linguagem fica não é o
caminho de instalação da aplicação, e sim o "Dados de Aplicativos".
Isso é ruim porque eu esperava poder distribuir um pacote com tudo
configurado.

O outro lado dessa situação é que não adianta alterar o arquivo xml
que contém as definições padrão de sintaxe das várias linguagens e
adicionar a definição da linguagem g-portugol, pois o menu de
linguagens do programa é "hardcoded" (!!) no código fonte, em uma
longa lista de #define's.

Por outro lado, no que tange a criar comandos, eu consegui definir
tranquilamente os comandos para compilar e executar os algoritmos,
inclusive com shortcuts de teclado, .

Voltando ao compilador, eu criei um novo build e enfrentei alguns
problemas com a codificação do texto, pois agora está tudo em UTF-8.
Sem ter conseguido encontrar nenhuma solução da API do Windows, acabei
fazendo um método ad-hoc para converter UTF-8 para latin-1 (e daí,
para o code page do console).

Como andei corringindo alguns bugs desde o lançamento do 1.0, acho que
jajá a gente lança uma revisão. Ao mesmo tempo, assim que eu ficar
minimamente satisfeito, vou ver se faço um tutorial para usar o
compilador com o notepad++.
