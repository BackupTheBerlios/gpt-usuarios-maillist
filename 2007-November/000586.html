* Abordagem #3 - Oferecer meios não tão especiais para manipular os
argumentos variáveis

Dado a seguinte estrutura:

estrutura Argumento
 tipo   : literal;
 valor : coringa;
fim-estrutura

E considerando-se que todo subprograma tem duas propriedades:
"argumentos" e "argumento" denotado pelos seguintes tipos:

argumentos  : inteiro;
argumento    : matriz[] de Argumento;

Dado um procedimetno p:

procedimento p( ... )
início
fim

podemos dizer que as seguintes propriedades existem:

f.argumentos
f.argumento

cujos valores são sensíveis ao contexto, só tendo valores válidos
dentro da própria função. Sendo assim, podemos escrever:

procedimento p( ... )
 i : inteiro;
início
 imprima("Total de argumentos passados: " + p.argumentos);

 para i de 0 até f.argumentos faça
   imprima("Tipo do argumento " + i + " : " + f.argumento[i].tipo;
   imprima("Valor do argumento " + i + " : " + f.argumento[i].valor;
fim

e a chamada p("ola", 10) imprimiria:

Tipo do argumento 0 : literal
Valor do argumento 0 : ola
Tipo do argumento 1 : inteiro
Valor do argumento 1 : 10


Acho que essa abordagem ficou mais atraente.
