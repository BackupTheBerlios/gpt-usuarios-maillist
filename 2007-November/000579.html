Por mim prossegue.

Não sei o que é ctor :-)

O que eu tinha pensado era, no início da geração de código da procedure que
declarou os dados (ou na procedure main no caso de dados globais), gerar
código em GptAssembly para inicializar esses dados.

Até daria para criar alguma otimização para dados globais. Como a gente vai
alocar uma área, a gente pode inicializar esse área com os dados corretos.
Com isso não é necessário gerar os comandos em GptAssembly.

Porém, para dados locais (que vão ser alocados na pilha) não tem como a
gente inicializar a área previamente. Com isso eu sugiro que a gente sempre
gere código para inicializar os dados, sejam eles globais ou
locais. Otimização não é meta do projeto por enquanto. Primeiro a gente
deixa tudo funcionando corretamente, e depois a gente otimiza :-)
