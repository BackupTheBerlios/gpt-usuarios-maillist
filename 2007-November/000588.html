Dei uma olhada rapida aqui em como algumas linguagens / compiladores
fazem testes automatizados. Todas as linguagens dinamicas que observei
utilizam testes unitários escritos na propria linguagem e acho que
mantem sua propria infra-estrutura. Creio que o mesmo vale para o
compilador do projeto free-pascal. Já o GCC utiliza o DejaGNU, que me
parece ser, no mínimo, uma bala de canhão tão difícil de digerir (ou
mais) quanto o gnu build system (autoconf/automake,etc), ainda mais
por utilizar _mais uma_ linguagem: uma tal de Expect.

Então, ainda não sei bem como poderiamos ter nossa infra-estrutura de testes.

Agora, quanto aos testes em si, para o front-end, estou pensando em
utilizar o antlr para criar uma gramática de descrição de testes.
Dessa forma, poderiamos descrever testes para comparar erros e geração
de código/AST utilizando uma notação, como:

//arquivo de descrição de testes

teste {
 código :
  "algoritmo bla;"

  ast: (algoritmo "bla")
  erros: [];
}

teste {
 código :
  "início x := 10 fim"

  ast: (inicio (:= "x" "10"))
  erros: [E01];  //E01: código do erro 'faltando vírgula após "10"'
}

--

É apenas um rascunho, mas a idéia é essa.
