A estrutura repita/enquanto tem uma armadilha que está me aborrecendo.

O seguinte código gera um erro de sintaxe (underline == identação):

repita
__//enunciados...
enquanto x < 10

Levando em consideração que temos uma esturutra "enquanto" na
gramática, este código é analisado da seguinte forma:

repita
__//enunciados
__enquanto x < 10

Portanto, o parser acusa falta do "faça" logo após o "10". Ele não vê
o "enquanto" como término da estrutura "repita", mas como o início de
outra estrutura de repetição.

Para resolver isso, imagino que temos que informar o parser para fazer
um "teste", verificando se existe um "faça" após a expressão da
condição, e devemos verificar se o contexto do código atual se
encontra dentro de um laço "repita". Mas isso não é tão trivial de se
fazer (acho até que requer uma pilha, para casos de loops dentro de
loops) e polui desagradavelmente o código do parser, que já é difícil
de deixar limpo. Pior, eu não sei o quão complexo ficará para
reportagem de erros (tanto o código quanto mensagens) para os N casos
de erros envolvendo estas duas estruturas.

Estou pensando em descartar esta estrutura, deixando apenas a repita/até.
Objeções?
