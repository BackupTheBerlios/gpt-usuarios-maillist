Pelo que eu saiba, tanto C como C++ geram a assinatura através de um símbolo

que representa a função. Algo como:

função f(i : inteiro) : inteiro => __funcao_f_inteiro
função f(i: literal)   : literal  => __funcao_f_literal

Uma solução simples seria acrescentar o retorno também na assinatura:

função f(i : inteiro) : inteiro => __funcao_f_inteiro_inteiro
função f(i: literal)   : literal  => __funcao_f_literal_literal

Mas isso está longe de ser perfeito porque pode existir ambiguidade no
retorno:

função f(i: inteiro) : inteiro;
função f(i:inteiro) : char;

variável x : real;

x := f(10); // qual invocar, ja que existe cast de inteiro para real e char
para real ???

Não sei bem porque C/C++ não inserem o retorno na assinatura. "Acho" que um
dos motivos
é que em C/C++ o usuário pode ignorar o retorno da função e isso geraria
ambiguidade na chamada.
